# Documentation Index

This directory is the repo-local system of record for implementation context that does not belong in `AGENTS.md`.

- `ARCHITECTURE.md` - package map, runtime workflow, and dependency boundaries.
- `QUALITY.md` - validation loop, offline testing expectations, library checks, and legibility checks.
- `RELEASING.md` - tag-first release process, release notes drafting, and PyPI publish trigger.

Keep `AGENTS.md` as a routing map. Put durable explanations here when future agents would otherwise need to rediscover project shape from code, including workflows such as Compendium Library publishing through `catalog.json`.
